================================================================================
FILE STRUCTURE
================================================================================

1. CALCULATION SCRIPTS (Python):
   
   calculate_ALL_PROCESSES.py
   - Calculates machine requirements for all 13 processes (A-M)
   - Based on Year +1 product demand (A1, A2, A3, B1, B2)
   - Uses BOM to calculate part demand (P1-P20)
   - Determines weekly workload per process
   - Computes machines needed (2-shift operation, 99% target utilization)
   - Output: 344 total machines

   optimized_layout_generator.py
   - Generates optimized facility layout
   - Arranges all 13 process areas left-to-right
   - Calculates grid configurations for each area
   - Creates layout visualization (PNG)
   - Exports layout summary (CSV with coordinates)
   - Layout: 1006 ft × 252 ft = 253,512 sq ft

   calculate_metrics.py
   - Calculates 4 key performance metrics:
     1. Distance per trip (rectilinear/Manhattan distance)
     2. Net utilization (weighted average)
     3. Cost of equipment
     4. Total area
   - Uses layout coordinates for distance calculations
   - Generates metrics summary report

2. OUTPUT FILES:

   Optimized_Layout_Summary.csv
   - Machine counts per process
   - Grid configurations (rows × cols)
   - X, Y coordinates for each area
   - Area dimensions (width × height)
   - Center coordinates for distance calculations
   - Layout summary (total width, height, area)

   Task3_Parts_Based_Layout.png
   - Visual representation of facility layout
   - Color-coded process areas
   - Machine counts displayed
   - Layout dimensions shown

   Metrics_Summary.txt
   - Summary of 4 key metrics
   - Detailed breakdown by process
   - Machine distribution table

3. DOCUMENTATION:

   INDEX.txt (this file)
   - File structure overview
   - Purpose of each file
   - Quick reference guide

   ANALYSIS_SUMMARY.txt
   - Complete analysis results
   - Detailed findings
   - Comparative analysis
   - Recommendations
